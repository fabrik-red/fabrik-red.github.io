Creating the image (disk.raw)

The fastest way of doing this (less than 1 hour “7min”) is to use AWS and chose the c4.8xlarge instance, at the moment of writhing this, is an instance with 36 vCPU / 60 RAM, the important thing to check for, is the amount of cores not really the memory. After finishing building, stop or terminate the build instance, mainly because is a very expensive instance and in best scenario you just want to do all this in less than 1 hour.

buildworld & buildkernel within a c4.8xlarge can be done in less than 7min.

Booting the instance

Boot using an existing FreeBSD AMI

Tip, search for Colin Percival AMI’s

The idea is to have a working FreeBSD AMI up and running the one will be just used for creating the new AMI.

svnlite

Once the instance is up and running, update the sources by doing:

svnlite co svn://svn.freebsd.org/base/stable/11 /usr/src

In this case we are using FreeBSD 11

*.conf & kernel

Edit file /etc/make.conf and if required the src.conf, to avoid any conflict with an existing configuration file we will use /etc/fabrik-src.conf and /etc/src-jail.conf for the jails.

fetch https://rawgit.com/fabrik-red/images/master/src.conf -o /etc/fabrik-src.conf
fetch https://rawgit.com/fabrik-red/images/master/src-jail.conf -o /etc/src-jail.conf
fetch https://rawgit.com/fabrik-red/images/master/make.conf -o /etc/make.conf

The kernel:

fetch https://rawgit.com/fabrik-red/images/master/fabrik.kernel -o /usr/src/sys/amd64/conf/FABRIK

Build -jX -j36

Build everything under /fabrik:

mkdir -p /fabrik/host
mkdir /fabrik/jail

Change to the /usr/src directory before building:

cd /usr/src

The world:

env MAKEOBJDIRPREFIX=/fabrik/host/obj SRCCONF=/etc/fabrik-src.conf make -j36 buildworld

The kernel:

env MAKEOBJDIRPREFIX=/fabrik/host/obj SRCCONF=/etc/fabrik-src.conf make -j36 buildkernel KERNCONF=FABRIK

The world for the jail:

env MAKEOBJDIRPREFIX=/fabrik/jail/obj SRCCONF=/etc/src-jail.conf make -j36 buildworld

Depending on how many cores set the value of -jN in this case N is 36 therefore we use -j36

zfs.sh

Create the image by running the zfs.sh script:

cd /fabrik
fetch https://rawgit.com/fabrik-red/images/master/zfs.sh -o /fabrik/zfs.sh
sh zfs.sh

After finish a disk.raw will be created and can be used as a base image.

AWS custom AMI

To create, upload the custom image is recommended to doit in US East (N. Virginia), for now, seems to be the only region where custom AMI’s could be uploaded, once you have tested your image and are happy with it, you can copy it to other AWS zones.

fabrik.sh

All in one script doing all the steps: fabrik.sh

fetch https://rawgit.com/fabrik-red/images/master/fabrik.sh -o /fabrik.sh
sh /fabrik.sh
