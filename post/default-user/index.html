Default username, password


login devops

devops is the only username created at boot time (firstboot) it has no password and always will try to obtain a new ssh public key from ssh-keys.online

If the instance is been used in AWS “key pairs” it will also try to fetch the public key selected when creating the image.

One the instance is up an running try to login:

$ ssh -l devops -i /path/to/private/ssh.key X.X.X.X

Or

$ssh devops@X.X.X.X -i /path/to/private/ssh.key

X.X.X.X is the IP of the server

If the server is not in AWS you will need to fetch the private key from ssh-keys.online.

Once the server is up and running visit https://ssh-keys.online/private/X.X.X.X in where X.X.X.X is the public IP of your instance, hopefully a private ssh key will be printed and is the one should be used for trying to login.

In case you have access to a terminal or VNC, the user root can be used.
